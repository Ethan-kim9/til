# 코딩 멋지게 하자.

1. 라이브러리를 자제해서 사용하자
   1. 간단한 기능을 하는 라이브러리면, 사용하지 않고 기능을 구현하자.
   2. 자신의 역량으로 충분히 개발 가능한 기능이 라이브러리를 사용할 경우, 큰 문제가 될 수 있음
2. 다른 사람의 코드를 자주 읽자.
3. 코드에 주석을 많이 작성하자.
4. 코딩 이외의 부가적인 것들은 최대한 하지 말자.
   1. 이상한거 꾸미고 시간낭비 하지말자
5. 컴포넌트 중심으로 코딩하자
   1. 모든 것을 컴포넌트로 나누고, 체계적으로 개발하면 이후 디버깅에 좋다.
6. 클린코드는 나중에 하자
   1. 프로젝트가 자리를 잡을 때 하는 것이 효율적이다.
7. 마크업을 최대한 체계적으로 작성하자
   1. 마크업은 로직을 구성하는 것만큼 중요하자
8. 최대한 사용자의 관점에서 바라보고 개발하자.
9. 값은 코드가 아닌 변수에 담아서 사용하자
